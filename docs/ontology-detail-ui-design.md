# 本体详情页 UI 设计方案

## 一、当前状态

### 已有组件
- `OntologyModeling.tsx` - 详情页主组件
- `OntologyGraph.tsx` - 图谱可视化组件
- `ontologyApi.ts` - API 服务层（完整）

### 已有 API 端点

| 端点 | 方法 | 功能 |
|------|------|------|
| `/ontologies/{id}` | GET | 获取本体基本信息 |
| `/ontologies/{id}/detail` | GET | 获取完整详情（含所有实体） |
| `/ontologies/{id}/classes` | GET/POST | 类 CRUD |
| `/ontologies/{id}/data-properties` | GET/POST | 数据属性 CRUD |
| `/ontologies/{id}/object-properties` | GET/POST | 对象属性 CRUD |
| `/ontologies/{id}/annotation-properties` | GET/POST | 注解属性 CRUD |
| `/ontologies/{id}/individuals` | GET/POST | 个体 CRUD |
| `/ontologies/{id}/publish` | POST | 发布本体 |
| `/ontologies/{id}/versions` | GET | 获取版本列表 |

## 三、Tab 功能设计

### 3.1 TBox 编辑器（默认 Tab）

**左侧面板 - 可折叠**
```
类列表 (Tree)
├── Person
│   ├── hasName (数据属性)
│   ├── hasAge (数据属性)
│   └── knows (对象属性)
└── Organization
    ├── name (数据属性)
    └── member (对象属性)
```

**中间面板 - 图谱可视化**
- D3.js 力导向图
- 支持缩放、平移
- 点击节点查看详情

**右侧面板 - 属性详情**
- 属性基本信息
- 定义域/值域
- **数据源映射配置** ← ETL 关键入口

## 八、API 补充需求

### 需要新增的 API

| API | 方法 | 说明 |
|-----|------|------|
| `/ontologies/{id}/classes/{classId}` | GET | 获取类详情 |

## 九、关键设计决策

1. **TBox/ABox 在同一页面，通过 Tab 切换** - 用户可以在编辑模式和查看模式间切换
2. **属性级 ETL 配置** - 每个属性独立配置数据源映射
3. **发布前置检查** - 确保 TBox 完整性和 ABox 可选的数据完整性
4. **版本快照** - 发布时自动创建不可变版本
