## 整改目标

1. OntoAgent 系统全面对接 Jena Graph Store Protocol

3. Layer 1 接口完整落地：50 个接口，后端 + 前端全部实现

---

## 命名图设计（已确认）

```
Fuseki Dataset: /onto-agent

每个本体（baseIri = http://onto-agent.com/ontology/{name}）

  {baseIri}/meta         → 本体元数据（owl:Ontology 自身）
  {baseIri}/tbox         → Schema（Class + DatatypeProperty + ObjectProperty）
  {baseIri}/abox         → 当前版本实例
  {baseIri}/abox@v1      → v1 快照（不可变）
  {baseIri}/abox@v2      → v2 快照（不可变）
```

---

## GSP / SPARQL 分工（已确认）

| 操作类型 | 使用协议 | 原因 |
|---------|---------|------|
| 全量追加 | GSP POST | 图级别追加，不支持条件 |
| 全量替换 | GSP PUT | 发布、快照回滚 |
| 全量读取 | GSP GET | 导出 OWL、读快照完整内容 |
| 整图删除 | GSP DELETE | 删除本体（级联删所有图）|
| 条件增删改 | SPARQL DELETE/INSERT | 更新单个实体、属性 |
| 条件查询 | SPARQL SELECT | 分页、过滤、聚合 |
| 创建快照 | SPARQL COPY | 复制 abox 到 abox@vN |

#### 完成的工作

**1. `jena_graph_protocol.py`（新增，380行）**
- `graph_get(uri)` → GET /data?graph=uri
- `graph_post(uri, data, content_type)` → POST /data?graph=uri
- `graph_put(uri, data, content_type)` → PUT /data?graph=uri
- `graph_delete(uri)` → DELETE /data?graph=uri
- `graph_exists(uri)` → HEAD 检查图是否存在
- `graph_get_turtle(uri)` → GET 指定接受 Turtle 格式
- `graph_post_empty(uri)` → POST 空图（初始化）
- `graph_put_empty(uri)` → PUT 空图（清空）
- `graph_ensure(uri, data)` → 确保图存在，不存在则创建
- `graph_copy(source, target)` → 读取源图内容 PUT 到目标图
- `graph_delete_all_version_snapshots(baseIri)` → 删除所有 abox@vN 快照
- `graph_delete_all_ontology_graphs(baseIri)` → 删除本体所有命名图（级联清理）

**2. `jena_version.py`（新增，250行）**
- `create_version_snapshot(ontologyIri, version)` → SPARQL COPY 当前 abox → abox@vN
- `rollback_to_version(ontologyIri, version)` → GET abox@vN → PUT abox
- `list_version_snapshots(ontologyIri)` → 列出所有 abox@vN 快照
- `delete_version_snapshot(ontologyIri, version)` → DELETE abox@vN
- `get_version_snapshot_content(ontologyIri, version)` → GET 快照全文
- `compare_versions(ontologyIri, vA, vB)` → 对比两个版本统计
- `publish_tbox(ontologyIri)` → 发布 TBox（读取草稿后 PUT 固化）
- `get_tbox_content(ontologyIri)` → 获取 TBox 全文
- `get_abox_content(ontologyIri)` → 获取 ABox 全文

**3. `jena_named_graph.py`（重构）**
- 移除与 GSP 层重复的方法（`copy_graph` → 改用 `graph_copy`，`delete_named_graph` → 改用 `graph_delete`）
- `insert_named_graph` 改为调用 `graph_post`
- `create_ontology_tbox` 改为写入 `{baseIri}/meta` 和 `{baseIri}/tbox`（与新命名图设计一致）
- 移除了 `{dataset}/{ontologyId}/tbox` 旧格式的支持

**4. `__init__.py`（更新）**
- 接入 `JenaGraphProtocolMixin` 和 `JenaVersionMixin`
- MRO 顺序：JenaBaseClient → JenaGraphProtocolMixin → JenaTBoxMixin → JenaABoxMixin → JenaNamedGraphMixin → JenaVersionMixin

## 决策记录

### 2026-04-11 决策

| 决策点 | 选项 | 结论 |
|--------|------|------|
| 命名图 URI 方案 | A: `{baseIri}/` vs B: `{dataset}/{ontologyId}/` | **A**：`{baseIri}/` 更直观 |
| meta 图是否独立 | A: 独立 vs B: 合并到 tbox | **A**：独立，查询效率更高 |
| Dataset 设计 | 共享 vs 独立 | 当前共用 `/onto-agent`，不变 |
| 快照不可变性 | 应用层强制 vs Fuseki 原生 | 应用层强制，不允许 PUT 快照图 |
| 草稿存储 | 独立图 vs 共用 tbox | 共用 tbox，PostgreSQL status 控制 |
