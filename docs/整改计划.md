# OntoAgent × Jena GSP 整改计划

> 创建时间：2026-04-11

---

## 整改目标

1. OntoAgent 系统全面对接 Jena Graph Store Protocol
2. 命名图设计：meta / tbox / abox / abox@vN 四层分离
3. Layer 1 接口完整落地：50 个接口，后端 + 前端全部实现

---

## 命名图设计（已确认）

```
Fuseki Dataset: /onto-agent

每个本体（baseIri = http://onto-agent.com/ontology/{name}）

  {baseIri}/meta         → 本体元数据（owl:Ontology 自身）
  {baseIri}/tbox         → Schema（Class + DatatypeProperty + ObjectProperty）
  {baseIri}/abox         → 当前版本实例
  {baseIri}/abox@v1      → v1 快照（不可变）
  {baseIri}/abox@v2      → v2 快照（不可变）
```

---

## GSP / SPARQL 分工（已确认）

| 操作类型 | 使用协议 | 原因 |
|---------|---------|------|
| 全量追加 | GSP POST | 图级别追加，不支持条件 |
| 全量替换 | GSP PUT | 发布、快照回滚 |
| 全量读取 | GSP GET | 导出 OWL、读快照完整内容 |
| 整图删除 | GSP DELETE | 删除本体（级联删所有图）|
| 条件增删改 | SPARQL DELETE/INSERT | 更新单个实体、属性 |
| 条件查询 | SPARQL SELECT | 分页、过滤、聚合 |
| 创建快照 | SPARQL COPY | 复制 abox 到 abox@vN |

## 决策记录

### 2026-04-11 决策

| 决策点 | 选项 | 结论 |
|--------|------|------|
| 命名图 URI 方案 | A: `{baseIri}/` vs B: `{dataset}/{ontologyId}/` | **A**：`{baseIri}/` 更直观 |
| meta 图是否独立 | A: 独立 vs B: 合并到 tbox | **A**：独立，查询效率更高 |
| Dataset 设计 | 共享 vs 独立 | 当前共用 `/onto-agent`，不变 |
| 快照不可变性 | 应用层强制 vs Fuseki 原生 | 应用层强制，不允许 PUT 快照图 |
| 草稿存储 | 独立图 vs 共用 tbox | 共用 tbox，PostgreSQL status 控制 |
| ETL 架构 | 属性声明来源 + 自动推断 join | **方案六**：每个属性声明 table/column/instance_id_column |
| System DB vs Business DB | — | System DB（5432）存所有系统表；Business DB（5433）为外部 ETL 数据源 |
